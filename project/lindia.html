Lindia

Automated Linear Regression Diagnostics in R

Context: I am a frequent R user for data manipulation and analysis work. In modeling, one simple technique that I always use is linear regression. To assess how well the linear regression fits the model, I often time need to take a quick skim of the regression diagnostics plot. However, the current syntax in base-R graphics (plotting features that comes with R) has made it very tedious to generate such plots, not to mention that crucial graphic types are not included.

I wanted to create a package that would allow users to generate diagnostics plot in a simple comment, while being able to retain the merit of grammar of graphics. Therefore, Lindia (short for linear diagnostics) was built as an extension to ggplot2 to provide streamlined plotting features of linear model diagnostic plots.

The package allows the user to use one command to output most of the graphs to access fitness of the linear regression to the data (e.g. residual plot, quantile-quantile plot, Cook’s distance plot), and arrange them to output in grid style.

Process: I was glad to have Statistics professor Sam Ventura to mentor this project. Writing package is quite different from any other experiences — we have to make sure that the package was robust enough to handle any malformed input, and that the output of the package does not differ much to users’ expectation.

After the package was done developing, we started the first round of dog fooding in the CMU Statistics department. One thing that stroke us as unexpected was that we didn’t account for the size of the regression people would feed into the lindia. Initially when we’re building the package, we tested it with a regression with only about 5 predictor variables, and so all the diagnostics plot were able to fit in a 4 by 3 grid. However, practically speaking, linear regression is used by data scientists to fit many predictor variables at once to assess which could potentially be useful in modeling. Therefore the plots that were cramped into the grid quickly gotten out of hand.

To remedy that problem, we added an additional feature that allowed user to specify the number of pages and plots per page in the output. That fixed the problem of diagnosting huge regression.

Challenges: There were quite a few unforeseen business challenges. For the first round of printing, I didn’t have a specific plan for quantity of postcards. I made an arbitrary decision of printing 40 postcards for each type, and 20 drawings in total. Because, in the end, I didn’t really know how people would receive it.

The package was submitted to the Comprehensive R Archive Network (CRAN) and was officially released as a package. We are hoping that lindia would be used in the Fall 17 semester in Advanced Data Analysis Course for hundreds of students to use.

The technical details of this package are included in this readme on Github.
